`Range`, `JSONContent`, `Schema`, `Node`, `Slice`, and `Decoration` are types from Tiptap or ProseMirror.

## `Change`

Represents one difference between documents.

### Properties

- `rangeA` (`Range`): The changed range in the original document.
- `rangeB` (`Range`): The matching range in the changed document.
- `isInlineGroup?` (`boolean`): Whether several inline changes were grouped together.
- `metadata?` (`ChangeMetadata`): Metadata associated with the change.

## `ChangeMetadata`

- `ChangeMetadata` (`Record<string, unknown>`): A shallow object containing arbitrary metadata for a change or step.

## Offset options

### `OffsetChangeOptions`

- `change` (`Change`): Change whose ranges are offset.
- `offsetA` (`number`): Value added to `rangeA` positions.
- `offsetB` (`number`): Value added to `rangeB` positions.

### `OffsetChangesOptions`

- `changes` (`Change[]`): Changes whose ranges are offset.
- `offsetA` (`number`): Value added to every `rangeA` position.
- `offsetB` (`number`): Value added to every `rangeB` position.

## `DiffOptions`

Controls document and fragment comparison.

### Properties

- `simplifyChanges?` (`boolean`): Merge nearby changes in inline mode. Default: `true`.
- `ignoreAttributes?` (`string[]`): Attributes to remove before comparing. Default: `['id', 'data-thread-id', '_hash']`.
- `ignoreMarks?` (`string[]`): Marks to ignore. Default: `['inlineThread']`.
- `ignoreNodes?` (`string[]`): Node types to unwrap or remove before comparing.
- `changeMergeDistance?` (`number | null`): Minimum unchanged length that separates adjacent changes. Default: `10`.
- `groupInlineChanges?` (`number`): Threshold for grouping inline changes in smart mode. Default: `1`.
- `mode?` (`'inline' | 'block' | 'smart'`): The comparison mode. Default: `'smart'`.
- `expandBlockChanges?` (`string[]`): Node types whose complete inline changes become block changes. Default: `['listItem']`.

## Comparison option types

### `CompareDocumentsOptions`

- `schema` (`Schema`): Schema used to parse both documents.
- `docA` (`JSONContent`): Original document.
- `docB` (`JSONContent`): Changed document.
- `diffOptions?` (`DiffOptions`): Comparison settings.

### `CompareFragmentsOptions`

- `schema` (`Schema`): Schema used to parse both fragments.
- `fragmentA` (`JSONContent[] | null`): Original fragment.
- `fragmentB` (`JSONContent[] | null`): Changed fragment.
- `diffOptions?` (`DiffOptions`): Comparison settings.

### `CompareStepsOptions`

- `stepSequence` (`StepSequence`): The document transitions to compare.
- `compareMetadata?` (`(a: ChangeMetadata, b: ChangeMetadata) => boolean`): Determines whether adjacent steps can be grouped.
- `diffOptions?` (`DiffOptions`): Comparison settings.
- `strictMode?` (`boolean`): Enables validation. Default: `false`.

### `CompareVersionsOptions`

- `versionA` (`Uint8Array`): Older Yjs v2 update.
- `versionB` (`Uint8Array`): Newer Yjs v2 update.
- `schema` (`Schema`): Schema used to hydrate the versions.
- `diffOptions?` (`DiffOptions`): Comparison settings.
- `compareMetadata?` (`(a: ChangeMetadata, b: ChangeMetadata) => boolean`): Determines whether adjacent changes can be grouped.
- `field?` (`string`): Yjs document field. Default: `'default'`.
- `permanentUserDataMapField?` (`string`): User-data map field. Default: `'__tiptapcollab__users'`.
- `enableDebugging?` (`boolean`): Enables debug logging. Default: `false`.

## `DiffSuggestion`

Represents one displayed change.

### Properties

- `id` (`string`): Unique suggestion ID.
- `range` (`Range`): Affected range in the editor.
- `content` (`Slice`): Replacement content.
- `metadata?` (`Record<string, any>`): Optional custom metadata.
- `displayOptions?` (`DisplayOptions`): Rendering options.
- `reviewMode?` (`'preview' | 'review'`): The review direction. Default: `'preview'`.
- `isInlineGroup?` (`boolean`): Whether inline changes were grouped.
- `mappableRange?` (`MappableRange`): Range tracked through transactions.

## `DisplayOptions`

Controls the default diff rendering.

### Properties

- `showReplacement?` (`boolean`): Shows replacement content. Default: `true`.
- `showSubChanges?` (`boolean`): Shows sub-changes in inline groups. Default: `true`.
- `attributes?` (`Record<string, any>`): Attributes on the original-content decoration.
- `replacementAttributes?` (`Record<string, any>`): Attributes on replacement content.
- `subChangeAttributes?` (`Record<string, any>`): Attributes on inline sub-changes.
- `replacementSubChangeAttributes?` (`Record<string, any>`): Attributes on sub-changes inside replacement content.
- `renderDecorations?` (`RenderDecorations`): Custom decoration renderer.

## `DisplayChangesOptions`

- `changes` (`Change[]`): Changes to display.
- `otherDoc` (`JSONContent`): The document used for replacement content.
- `displayOptions?` (`DisplayOptions`): Rendering options.
- `reviewMode?` (`'preview' | 'review'`): Review direction. Default: `'preview'`.

## `CompareOptions` and `CompareStorage`

- `CompareOptions` (`Record<string, never>`): Compare has no configuration options.
- `CompareStorage.pluginKey` (`PluginKey<ComparePluginState>`): The ProseMirror plugin key used to manage diff suggestions.

## `DefaultRenderDecorationsOptions`

- `showMainDecorations?` (`boolean`): Shows the original-content and sub-change decorations. Default: `true`.
- `showReplacement?` (`boolean`): Shows replacement content. Defaults to `DisplayOptions.showReplacement`.
- `showSubChanges?` (`boolean`): Shows sub-changes. Defaults to `DisplayOptions.showSubChanges`.
- `attributes?` (`Record<string, any>`): Attributes on original-content decorations.
- `replacementAttributes?` (`Record<string, any>`): Attributes on replacement content.
- `subChangeAttributes?` (`Record<string, any>`): Attributes on inline sub-changes.
- `replacementSubChangeAttributes?` (`Record<string, any>`): Attributes on sub-changes inside replacement content.

## `RenderDecorationsOptions`

- `range` (`Range`): Range occupied by the diff suggestion.
- `isSelected` (`boolean`): Whether the suggestion is selected.
- `diffSuggestion` (`DiffSuggestion`): Suggestion being rendered.
- `defaultRenderDecorations` (`(options?: DefaultRenderDecorationsOptions) => Decoration[]`): Renders the default decorations.

`RenderDecorations` is a function that accepts `RenderDecorationsOptions` and returns `Decoration[]`.

## `AllDiffSuggestionsOptions`

- `filter?` (`(diffSuggestion: DiffSuggestion) => boolean`): Selects suggestions for an accept-all or reject-all command.

## Step sequences

### `Step`

- `before` (`Node`): Document before the step.
- `after` (`Node`): Document after the step.
- `metadata` (`ChangeMetadata`): Metadata associated with the step.
- `changes` (`Change[] | null`): Cached changes, when available.
- `changedArea` (`Change | null`): Cached changed area, when available.

### `AddStepOptions`

- `doc` (`Node`): Document after the new step.
- `metadata` (`ChangeMetadata`): Step metadata.
- `changes` (`Change[] | null`): Cached changes.
- `changedArea?` (`Change | null`): Cached changed area.

### `StepSequenceOptions`

- `before` (`Node`): Initial document.
- `docs?` (`Node[]`): Documents after each recorded step.
- `metadata?` (`ChangeMetadata[]`): Metadata for each step.
- `changes?` (`Array<Change[] | null>`): Cached changes for each step.
- `changedAreas?` (`Array<Change | null>`): Cached changed areas for each step.

### `StepSequenceJSON`

- `before` (`JSONContent`): Serialized initial document.
- `docs` (`JSONContent[]`): Serialized documents after each step.
- `metadata` (`ChangeMetadata[]`): Metadata for each step.
- `changes` (`Array<Change[] | null>`): Cached changes for each step.
- `changedAreas` (`Array<Change | null>`): Cached changed areas for each step.

### `StepSequenceFromJSONOptions`

- `json` (`StepSequenceJSON`): Serialized sequence.
- `schema` (`Schema`): Schema used to hydrate the documents.

## Mappable ranges

### `MappablePosition`

- `position` (`number`): Current document position.
- `update` (`(transaction: Transaction) => MappablePositionUpdateResult`): Maps the position through a transaction.

### `MappablePositionUpdateResult`

- `mappablePosition` (`MappablePosition`): Updated position.
- `mapResult` (`MapResult | null`): ProseMirror mapping result.

### `MappableRangeUpdateResult`

- `mappableRange` (`MappableRange`): Updated range.
- `mapResultFrom` (`MapResult | null`): Mapping result for the start position.
- `mapResultTo` (`MapResult | null`): Mapping result for the end position.

`MappableRange` is constructed with `editor` (`Editor`) and `range` (`Range`). Its `update(transaction)` method returns `MappableRangeUpdateResult`; `getRange()` returns `Range`.

## Version and schema helpers

### `GetDocumentFromVersionOptions`

- `version` (`Uint8Array`): Complete Yjs v2 update.
- `schema` (`Schema`): Schema used to hydrate the document.
- `field?` (`string`): Yjs document field. Default: `'default'`.

### `SerializedSchema`

- `topNode?` (`string`): Top node name.
- `nodes` (`Record<string, SerializedNodeSpec>`): Serialized node specs by name.
- `marks` (`Record<string, SerializedMarkSpec>`): Serialized mark specs by name.

`SerializedAttributeSpec`, `SerializedNodeSpec`, and `SerializedMarkSpec` are JSON-safe versions of their matching ProseMirror schema specifications. Each preserves its serializable properties; the `attrs` property is `Record<string, SerializedAttributeSpec>` when present.
