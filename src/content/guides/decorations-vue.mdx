In this tutorial, you will build find and replace without changing the document just to show search results. The finished feature uses all three decoration types:

- **Inline decorations** highlight every match.
- A **node decoration** outlines the block containing the active match.
- A **widget decoration** renders a Vue replacement button beside the active match.

The search controls live outside the editor. Extension storage holds their current values, and `updateDecorations()` refreshes the view whenever those values change.

## Create the replacement widget

`VueWidgetRenderer` adds `editor` and `getPos` to your component props. Vue decoration components must render a single root element.

```vue
<!-- ReplaceButton.vue -->
<template>
  <button class="search-replace-button" contenteditable="false" type="button" @click="replace">
    Replace
  </button>
</template>

<script setup lang="ts">
import type { Editor } from '@tiptap/core'

const props = defineProps<{
  editor: Editor
  getPos: () => number | undefined
  replacement: string
  searchTerm: string
}>()

const replace = () => {
  const to = props.getPos()

  if (to === undefined || !props.searchTerm) {
    return
  }

  props.editor.commands.insertContentAt(
    { from: to - props.searchTerm.length, to },
    props.replacement,
  )
  props.editor.commands.updateDecorations('searchAndReplace')
}
</script>
```

<Callout title="Always read the current position" variant="info">
  Keep the `getPos` function, not its initial result. Tiptap updates the function as edits map the
  widget to a new document position.
</Callout>

## Create the search extension

The extension searches through text nodes without changing them. It uses the `manual` update strategy. This means the toolbar decides when to run the search again.

```ts
// search-and-replace.ts
import { decoration, Extension } from '@tiptap/core'
import type { DecorationDescriptor } from '@tiptap/core'
import { VueWidgetRenderer } from '@tiptap/vue-3'

import ReplaceButton from './ReplaceButton.vue'

interface SearchStorage {
  activeIndex: number
  replacement: string
  searchTerm: string
}

declare module '@tiptap/core' {
  interface Storage {
    searchAndReplace: SearchStorage
  }
}

interface Match {
  from: number
  to: number
}

export const SearchAndReplace = Extension.create<{}, SearchStorage>({
  name: 'searchAndReplace',

  addStorage() {
    return {
      activeIndex: 0,
      replacement: '',
      searchTerm: '',
    }
  },

  addDecorations() {
    return {
      update: 'manual',
      create: ({ editor, state }) => {
        const { activeIndex, replacement, searchTerm } = editor.storage.searchAndReplace
        const term = searchTerm.trim().toLocaleLowerCase()

        if (!term) {
          return []
        }

        const matches: Match[] = []

        state.doc.descendants((node, pos) => {
          if (!node.isText || !node.text) {
            return
          }

          const text = node.text.toLocaleLowerCase()
          let index = text.indexOf(term)

          while (index !== -1) {
            matches.push({
              from: pos + index,
              to: pos + index + term.length,
            })
            index = text.indexOf(term, index + term.length)
          }
        })

        if (!matches.length) {
          return []
        }

        const normalizedIndex = ((activeIndex % matches.length) + matches.length) % matches.length
        const activeMatch = matches[normalizedIndex]
        const decorations: DecorationDescriptor[] = matches.map((match, index) =>
          decoration.inline(match.from, match.to, {
            class: index === normalizedIndex ? 'search-match search-match-active' : 'search-match',
          }),
        )

        const $active = state.doc.resolve(activeMatch.from)
        const blockDepth = Math.max(1, $active.depth)

        decorations.push(
          decoration.node($active.before(blockDepth), $active.after(blockDepth), {
            class: 'search-active-block',
          }),
          VueWidgetRenderer(ReplaceButton, {
            editor,
            pos: activeMatch.to,
            key: 'search-active-match',
            props: { replacement, searchTerm: term },
            side: 1,
          }),
        )

        return decorations
      },
    }
  },
})
```

The widget key is a constant because there is exactly one active-match widget. Its identity stays stable while it moves between results, allowing Vue to reuse the component.

## Add the editor and search controls

Keep the input fields in Vue refs. When a value or the active result changes, copy it into extension storage and force a decoration rebuild. Destroying the editor on unmount also cleans up the widget renderer.

```vue
<!-- TiptapEditor.vue -->
<template>
  <div>
    <div class="search-toolbar">
      <input v-model="searchTerm" aria-label="Find" />
      <input v-model="replacement" aria-label="Replace with" />
      <button type="button" @click="refresh(0)">Find</button>
      <button type="button" @click="previous">Previous</button>
      <button type="button" @click="next">Next</button>
      <button type="button" @click="clear">Clear</button>
    </div>
    <EditorContent :editor="editor" />
  </div>
</template>

<script setup lang="ts">
import StarterKit from '@tiptap/starter-kit'
import { Editor, EditorContent } from '@tiptap/vue-3'
import { onBeforeUnmount, ref } from 'vue'

import { SearchAndReplace } from './search-and-replace'
import './styles.css'

const replacement = ref('editor')
const searchTerm = ref('tiptap')
const editor = new Editor({
  extensions: [StarterKit, SearchAndReplace],
  content: `
    <h2>Search this Tiptap document</h2>
    <p>Tiptap makes editor interfaces approachable.</p>
    <p>Try replacing either Tiptap match.</p>
  `,
})

const refresh = (activeIndex = 0) => {
  Object.assign(editor.storage.searchAndReplace, {
    activeIndex,
    replacement: replacement.value,
    searchTerm: searchTerm.value,
  })
  editor.commands.updateDecorations('searchAndReplace')
}

const previous = () => refresh(editor.storage.searchAndReplace.activeIndex - 1)
const next = () => refresh(editor.storage.searchAndReplace.activeIndex + 1)
const clear = () => {
  searchTerm.value = ''
  editor.storage.searchAndReplace.searchTerm = ''
  editor.commands.updateDecorations('searchAndReplace')
}

onBeforeUnmount(() => {
  editor.destroy()
})
</script>
```

## Style the three decoration types

```css
.search-match {
  background: #fff3a3;
}

.search-match-active {
  background: #ffc857;
}

.search-active-block {
  outline: 2px solid #ffc857;
  outline-offset: 3px;
}

.search-replace-button {
  margin-inline-start: 0.25rem;
  border: 0;
  border-radius: 0.25rem;
  padding: 0.125rem 0.375rem;
  background: #1f2937;
  color: white;
}
```

You now have a view-only search layer and a replacement action that changes content only when the user asks it to. For production use, consider case-sensitivity and whole-word options, keyboard shortcuts, scrolling the active result into view, and searches that span multiple text nodes.
