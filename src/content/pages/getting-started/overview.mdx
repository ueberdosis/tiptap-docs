## Limitations

### Non-Splittable Content Blocks

The pagination system works by using CSS floats to position elements around page gaps. This approach has specific requirements for how content elements must behave.

**The Issue:** Content blocks need to flow naturally around page boundaries. This works correctly when a block either:

- Flows naturally with the page layout
- Is a non-splittable block (such as tables, figures, or certain styled containers) but fits completely within a single page

Problems occur when a non-splittable block grows too large to fit on a single page. Since these blocks cannot be broken across pages by the float-based pagination, they will never fit properly, causing the pagination layout to break.

> **Technical Note:** Non-splittable blocks are typically elements that create a [Block Formatting Context (BFC)](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_display/Block_formatting_context), which affects how they interact with floated elements.

**Solutions:**

1. **Set Maximum Height:** Apply a `max-height` to large non-splittable blocks that ensures they fit within a single page. This prevents elements from exceeding page boundaries. You can use the `--page-max-height` CSS property to ensure your element does not exceed a page boundary.

2. **Manual Node Splitting:** For content that needs to span multiple pages, split it manually into smaller nodes. Note that this approach introduces structural changes to your content and may affect the document's semantic structure.

When designing custom nodes or styling content, consider these limitations to ensure proper pagination behavior.
