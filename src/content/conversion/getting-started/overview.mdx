Tiptap Conversion translates between document formats and your editor's document model. It supports importing DOCX and Markdown, and exporting to DOCX, PDF, ODT, EPUB, and Markdown. It is not a document renderer. It is a bridge. What renders in your editor depends on which extensions you have installed.

<Callout title="Pro package" variant="hint">
  Conversion is a Pro package included with all [Tiptap subscriptions](https://tiptap.dev/pricing). Before installation, you'll need to set up [access to Tiptap's private NPM
  registry](/guides/pro-extensions).
</Callout>

## How conversion works

Conversion is a multi-step pipeline, not a single operation. The conversion service handles the translation between file formats and Tiptap's JSON document model. Your editor handles the rendering.

```
  Source                Conversion             Tiptap              Your Editor             Export              Output
  document              Service                JSON                + Extensions                                formats

   │                    │                      │                     │                      │                 │
   │    ── import ──▶   │   ── produces ──▶    │   ── render ──▶     │   ── export ──▶      │  ── output ──▶  │
   │                    │                      │                     │                      │                 │
                  Tiptap handles this                         You handle this
```

1. **Import:** The conversion service parses your source document and produces Tiptap JSON, a structured representation of the document's content, formatting, and layout.
2. **Editor rendering:** Your Tiptap editor receives the JSON. For each node type in the JSON (heading, table, image, etc.), the editor needs a matching extension installed to render it. Content without a matching extension will not appear in the editor.
3. **Export:** The export extension converts your editor's content back into a document format. It exports what the editor currently contains, not what the original source document had.

<Callout variant="warning" title="The key thing to understand">
  Your editor only renders content it has extensions for. If the conversion service produces a node type that your editor does not recognize, that content will not appear. Check the [Content types](/conversion/content-types/overview) pages to see which extensions you need for the content your documents use.
</Callout>

## Two levels of integration

How much of the document experience you recreate depends on what you are building.

### Content-level: core extensions

For most content like text, headings, lists, tables, images, links, and formatting, standard Tiptap extensions handle the rendering. `StarterKit` covers the basics. Add `TableKit`, `Image`, `TextAlign`, and text styling extensions for broader coverage.

This is enough if you are building a **content editor** or **CMS** that accepts document uploads. The text content renders correctly in a continuous-scroll editor. For Markdown imports, core extensions are typically all you need.

### Document-level: Pages extension

For page-aware documents with headers, footers, page breaks, page numbering, page size and margins, you need the [Pages extension](/pages/getting-started/overview). It provides a paginated editing experience where content flows across visual pages, and headers and footers have dedicated editing areas.

This is what you need if you are building a **document editor** where users expect a page layout. Without Pages, imported headers and footers have nowhere to render, page breaks are ignored, and there is no concept of page boundaries. This applies primarily to DOCX imports, as Markdown does not carry page layout information.

| What you're building | What you need |
| --- | --- |
| Content editor / CMS with document upload | Core extensions (StarterKit + formatting) |
| Document editor with page layout | Core extensions + Pages extension |
| Headless pipeline (no editor) | REST API only, no extensions needed |

## What "supported" means

Not every document feature works the same way through the pipeline. When we say a feature is "supported," it falls into one of two tiers:

- **Supported:** The conversion service produces it and an official Tiptap extension exists to render it. You install the extension, configure it, and the content works across import, rendering, and export. The [Content types](/conversion/content-types/overview) pages show which extensions you need and how to configure them for each feature.
- **Conversion only:** The conversion service produces JSON for it, but no official extension exists to render it yet. Without a matching extension, this content will not appear in the editor. You can build a custom extension to handle it, or choose not to render it.

See the [Supported features](/conversion/getting-started/feature-support-matrix) page for the full breakdown by content type.

## Two ways to integrate

- **Editor extensions:** Import and export directly within your Tiptap editor instance. The extensions handle the conversion pipeline as editor commands.
- **REST API:** Use HTTP endpoints to convert documents without a Tiptap editor. Useful for server-side processing, background jobs, or non-editor integrations.

## Legacy Import/Export extensions

The legacy `@tiptap-pro/extension-import` and `@tiptap-pro/extension-export` packages are being deprecated and will be sunset in 2026. These extensions receive no new features and are maintained for existing users only.

- [Legacy DOCX Import/Export](/conversion/legacy/overview)
- [Legacy ODT Import/Export](/conversion/legacy/odt/editor-extensions)
- [Legacy Markdown Import/Export](/conversion/legacy/markdown/editor-extensions)

**Migrating to the new extensions:** The current [Import DOCX](/conversion/import/docx/editor-import) and [Export DOCX](/conversion/export/docx/editor-export) extensions provide improved conversion quality, more format options, and active development. We recommend migrating as soon as possible.
