## What you can try

**Import** a `.docx` file using the upload button. The conversion service parses the document and produces Tiptap JSON that the editor renders. Text formatting, headings, tables, images, lists, and links are all preserved.

**Edit** the imported content directly in the editor. Add text, change formatting, insert new elements. The editor supports everything the import produces.

**Export** back to DOCX using the export button. The export extension converts the editor's current content into a Word-compatible file that you can open in Microsoft Word, Google Docs, or any other DOCX-compatible application.

## What to look for

- **Text formatting** carries through: bold, italic, underline, strikethrough, text color, highlight, font family, and font size all survive the round-trip.
- **Document structure** is preserved: headings maintain their hierarchy, lists keep their nesting, tables retain their layout with merged cells and column widths.
- **Images** are extracted from the DOCX, uploaded to your configured endpoint, and embedded with their original dimensions.
- **What changes:** Paragraph-level styling like spacing, indentation, and Word theme colors are not rendered in the editor. The exported DOCX uses Tiptap's default styles, not the original document's styles. See [Styling converted content](/conversion/getting-started/guides/styling-converted-content) for how to control this.

## Supported formats

This demo shows the DOCX round-trip. The conversion pipeline also supports exporting to other formats. Each format generates DOCX first, then converts to the target format via a cloud service:

- **[PDF](/conversion/export/pdf/editor-extension)** for print-ready documents
- **[ODT](/conversion/export/odt/editor-extension)** for OpenDocument-compatible applications
- **[EPUB](/conversion/export/epub/editor-extension)** for e-readers
- **[DOC](/conversion/export/doc/editor-extension)** for legacy Word compatibility
- **[Markdown](/conversion/export/markdown/editor-extension)** for text-based workflows (client-side, independent of the DOCX pipeline)
