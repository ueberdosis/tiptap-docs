Text color and highlight convert between Word's color model and Tiptap's `textStyle` and `highlight` marks. Both round-trip when the right extensions are installed.

## What you need

- **Extensions:** `@tiptap/extension-text-style`, `@tiptap/extension-color`, and `@tiptap/extension-highlight` (none are in StarterKit)
- **Configuration:** `Highlight.configure({ multicolor: true })`

```ts
import TextStyle from '@tiptap/extension-text-style'
import Color from '@tiptap/extension-color'
import Highlight from '@tiptap/extension-highlight'

const editor = new Editor({
  extensions: [
    StarterKit,
    TextStyle,
    Color,
    Highlight.configure({ multicolor: true }),
  ],
})
```

<Callout title="multicolor: true is required" variant="warning">
  Without `multicolor: true`, the Highlight extension only supports a single generic highlight. Different highlight colors from imported DOCX documents are not preserved, and every highlight renders as the default yellow.
</Callout>

## Support overview

| | Import | Editor | Export |
| --- | --- | --- | --- |
| Text color | Supported (`textStyle` mark) | Supported (TextStyle + Color) | Supported (normalized to hex) |
| Highlight | Supported (`highlight` mark) | Supported (Highlight, multicolor) | Supported (named or hex) |
| Shading (`<w:shd>`) | Supported (maps to `highlight`) | Supported | Supported (solid shading) |
| Theme colors | Not supported (not converted) | Not applicable | Not applicable |

## Import

Import text color and highlight using the [editor extension](/conversion/import/docx/editor-extension) or the [REST API](/conversion/import/docx/rest-api). Both produce identical output.

<CodeDemo isPro path="/Extensions/ContentTypeTextColorHighlightImport?hideSource=true" />

The conversion service maps Word color elements to Tiptap marks:

| Word element | Tiptap mark | Attribute |
| --- | --- | --- |
| `<w:color val="FF0000">` | `textStyle` | `{ color: "#FF0000" }` |
| `<w:highlight val="yellow">` | `highlight` | `{ color: "yellow" }` |
| `<w:shd val="clear" fill="00FFFF">` | `highlight` | `{ color: "#00FFFF" }` |

Both `<w:highlight>` and `<w:shd>` produce the same `highlight` mark. The developer does not need to distinguish between them.

<Callout title="Theme colors are not imported" variant="warning">
  Word theme colors (such as `accent1`, `accent2`, or `themeColor` attributes on `<w:color>`) are not converted during import. If a text run uses only a theme color with `val="auto"`, the color is not included in the output. Only explicit hex color values in `<w:color val="...">` are preserved.
</Callout>

## Editor rendering

The [`Color`](/editor/extensions/marks/color) extension adds color support to [`TextStyle`](/editor/extensions/marks/text-style), and the [`Highlight`](/editor/extensions/marks/highlight) extension handles background colors. All must be installed separately from StarterKit.

<CodeDemo isPro path="/Extensions/ContentTypeTextColorHighlightEditor?hideSource=true" />

Text color renders as an inline `style="color: ..."` on a `<span>` element. Highlight renders as a `<mark>` element with a `background-color` style.

Without the Color extension, text color attributes from import are not recognized by the schema. Without Highlight (or without `multicolor: true`), highlight colors are not preserved. See the [invalid schema guide](/guides/invalid-schema) for how to manage unrecognized marks and attributes.

## Export

Export text color and highlight using the [editor extension](/conversion/export/docx/editor-extension) or the [REST API](/conversion/export/docx/rest-api). Both handle color and highlight identically.

<CodeDemo isPro path="/Extensions/ContentTypeTextColorHighlightExport?hideSource=true" />

On export, color values are normalized to 6-digit hex. The following input formats are accepted:

- Hex with hash: `#FF0000`, `#F00`, `#FF0000AA`
- RGB/RGBA: `rgb(255, 0, 0)`, `rgba(255, 0, 0, 0.5)`
- Named CSS colors: `red`, `blue`, `green`, `yellow`, `cyan`, `purple`, and 14 others

Highlights export as either a named Word highlight color (when the color matches a Word enum value like `yellow` or `cyan`) or as solid character shading with a hex fill value. A highlight mark with no color attribute defaults to yellow.
