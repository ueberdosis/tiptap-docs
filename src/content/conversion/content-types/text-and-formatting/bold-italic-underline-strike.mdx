These are inline marks applied to text runs. In Word they come from run properties (`<w:rPr>`). In Tiptap they are marks on text nodes. All four are included in StarterKit.

## What you need

- **Extensions:** `StarterKit` (includes Bold, Italic, Underline, and Strike)
- **Configuration:** None required. All four marks work out of the box.

## Support overview

| Mark | Import | Editor | Export | Notes |
| --- | --- | --- | --- | --- |
| Bold | Supported | Supported (StarterKit) | Supported | Full round-trip |
| Italic | Supported | Supported (StarterKit) | Supported | Full round-trip |
| Underline | Supported (`val="single"` only) | Supported (StarterKit) | Supported | Other underline styles not converted |
| Strikethrough | Supported | Supported (StarterKit) | Supported | Full round-trip |
| Double strikethrough | Not supported | Not supported | Not supported | Not converted |

## Import

Import these marks using the [editor extension](/conversion/import/docx/editor-extension) or the [REST API](/conversion/import/docx/rest-api). Both produce identical output.

<CodeDemo isPro path="/Extensions/ContentTypeBoldItalicUnderlineStrikeImport?hideSource=true" />

The conversion service reads Word run properties and produces the corresponding Tiptap marks:

| Word element | Tiptap mark |
| --- | --- |
| `<w:b/>` | `bold` |
| `<w:i/>` | `italic` |
| `<w:u val="single"/>` | `underline` (other `val` values not converted) |
| `<w:strike/>` | `strike` |
| `<w:dstrike/>` | Not handled |

Bold, italic, and strikethrough import cleanly. Underline is only imported when the Word element has `val="single"`. Other underline styles (double, wavy, dotted, dash) are not converted. Combined marks (bold + italic on the same text) are fully preserved.

## Editor rendering

All four marks are included in [`StarterKit`](/editor/extensions/functionality/starterkit). No separate installation is needed.

<CodeDemo isPro path="/Extensions/ContentTypeBoldItalicUnderlineStrikeEditor?hideSource=true" />

| Mark | Extension | Renders as | Keyboard shortcut |
| --- | --- | --- | --- |
| bold | [`Bold`](/editor/extensions/marks/bold) | `<strong>` | Cmd+B |
| italic | [`Italic`](/editor/extensions/marks/italic) | `<em>` | Cmd+I |
| underline | [`Underline`](/editor/extensions/marks/underline) | `<u>` | Cmd+U |
| strike | [`Strike`](/editor/extensions/marks/strike) | `<s>` | Cmd+Shift+S |

All four marks can be active simultaneously on the same text.

## Export

Export these marks using the [editor extension](/conversion/export/docx/editor-extension) or the [REST API](/conversion/export/docx/rest-api). Both handle them identically.

<CodeDemo isPro path="/Extensions/ContentTypeBoldItalicUnderlineStrikeExport?hideSource=true" />

Each mark maps to a DOCX run property:

| Tiptap mark | DOCX run property |
| --- | --- |
| `bold` | `bold = true` |
| `italic` | `italics = true` |
| `underline` | `UnderlineType.SINGLE` |
| `strike` | `strike = true` |

Strikethrough round-trips correctly. The import produces the correct `strike` mark name, so no formatting is lost.

## Strikethrough

<Callout title="Double strikethrough is not supported" variant="info">
  Double strikethrough (`<w:dstrike/>`) is not handled. It is not converted on import.
</Callout>

<Callout title="Only single underline is imported" variant="warning">
  Word supports many underline styles (double, wavy, dotted, dash, and others). On import, only `val="single"` produces an `underline` mark. All other styles are not converted and are not downgraded to a plain underline. On export, every underline becomes `UnderlineType.SINGLE`. Text content is preserved, but the underline formatting is lost for non-single styles.
</Callout>
