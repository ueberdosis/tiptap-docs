Subscript and superscript marks import from DOCX and render in the editor, but are not preserved on export. Content like H₂O will import and display correctly but export as plain H2O.

## What you need

- **Extensions:** `@tiptap/extension-subscript` and `@tiptap/extension-superscript` (not in StarterKit, must install separately)
- **Configuration:** None required beyond installation.

## Support overview

| | Import | Editor | Export |
| --- | --- | --- | --- |
| Subscript | Supported | Supported (separate extension) | Not supported |
| Superscript | Supported | Supported (separate extension) | Not supported |

## Import

Import subscript and superscript using the [editor extension](/conversion/import/docx/editor-extension) or the [REST API](/conversion/import/docx/rest-api). Both produce identical output.

<CodeDemo isPro path="/Extensions/ContentTypeSubscriptSuperscriptImport?hideSource=true" />

The conversion service reads `<w:vertAlign>` run properties and produces the corresponding marks:

| Word element | Tiptap mark |
| --- | --- |
| `<w:vertAlign val="subscript">` | `subscript` |
| `<w:vertAlign val="superscript">` | `superscript` |

Chemical formulas (H₂O, CO₂), mathematical notation (x², E=mc²), and footnote references all import correctly.

## Editor rendering

The [`Subscript`](/editor/extensions/marks/subscript) and [`Superscript`](/editor/extensions/marks/superscript) extensions must be installed separately. They are not part of StarterKit.

<CodeDemo isPro path="/Extensions/ContentTypeSubscriptSuperscriptEditor?hideSource=true" />

```bash
npm install @tiptap/extension-subscript @tiptap/extension-superscript
```

```ts
import Subscript from '@tiptap/extension-subscript'
import Superscript from '@tiptap/extension-superscript'

const editor = new Editor({
  extensions: [
    StarterKit,
    Subscript,
    Superscript,
  ],
})
```

Subscript renders as `<sub>`, superscript renders as `<sup>`. Keyboard shortcuts: Cmd+, for subscript, Cmd+. for superscript.

<Callout title="Without these extensions, imported marks are dropped" variant="info">
  If you do not install the Subscript and Superscript extensions, the marks from import will not be recognized by the editor schema. See the [invalid schema guide](/guides/invalid-schema) for how to manage unrecognized marks.
</Callout>

## Export

Neither the [editor extension](/conversion/export/docx/editor-extension) nor the [REST API](/conversion/export/docx/rest-api) exports subscript or superscript marks.

<CodeDemo isPro path="/Extensions/ContentTypeSubscriptSuperscriptExport?hideSource=true" />

<Callout title="Subscript and superscript are not exported" variant="danger">
  The DOCX export converter does not handle `subscript` or `superscript` marks. They are not converted during export. Content like H₂O will appear as plain H2O in the exported document. This affects chemical formulas, mathematical notation, footnote references, and any other use of sub/superscript text.
</Callout>

Subscript and superscript applied in the editor work correctly for display. The limitation only affects export to DOCX.

Subscript and superscript within math or LaTeX expressions follow a separate code path (using OOXML math elements `m:sSub` and `m:sSup`) and are not affected by this limitation.
