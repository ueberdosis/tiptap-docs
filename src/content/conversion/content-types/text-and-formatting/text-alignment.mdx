Text alignment is a paragraph-level attribute that round-trips between DOCX and the editor. Configure the TextAlign extension and alignment is preserved through import, editing, and export.

## What you need

- **Extension:** `@tiptap/extension-text-align` (not in StarterKit, must be installed separately)
- **Configuration:** `TextAlign.configure({ types: ['paragraph', 'heading'] })`
- **That is it.** No additional setup needed for alignment to work end-to-end.

## Support overview

| | Import | Editor | Export |
| --- | --- | --- | --- |
| Left | Supported | Supported | Supported |
| Center | Supported | Supported | Supported |
| Right | Supported | Supported | Supported |
| Justify | Supported | Supported | Supported |

## Import

Import text alignment using the [editor extension](/conversion/import/docx/editor-extension) or the [REST API](/conversion/import/docx/rest-api). Both produce identical output.

<CodeDemo isPro path="/Extensions/ContentTypeTextAlignmentImport?hideSource=true" />

The conversion service reads the `<w:jc>` element from each DOCX paragraph and maps it to a `textAlign` attribute on the corresponding Tiptap node.

| Word `<w:jc>` value | Tiptap `textAlign` |
| --- | --- |
| `left` | `left` |
| `center` | `center` |
| `right` | `right` |
| `both` | `justify` |
| `start` | `start` (passed through, may not be recognized by TextAlign) |
| `end` | `end` (passed through, may not be recognized by TextAlign) |

The `textAlign` attribute is set on both `paragraph` and `heading` nodes.

## Editor rendering

The [`TextAlign`](/editor/extensions/functionality/text-align) extension adds `textAlign` as a global attribute on the node types you specify. It renders as an inline `style="text-align: ..."` on the HTML element.

<CodeDemo isPro path="/Extensions/ContentTypeTextAlignmentEditor?hideSource=true" />

```ts
import TextAlign from '@tiptap/extension-text-align'

TextAlign.configure({
  types: ['paragraph', 'heading'],
})
```

<Callout title="Silent failure without configuration" variant="warning">
  The TextAlign extension's `types` array is empty by default. If you forget to add `paragraph` and `heading`, the `textAlign` attribute from import is not recognized by the schema and will not be applied. No error is thrown. See the [invalid schema guide](/guides/invalid-schema) for more on this behavior.
</Callout>

## Export

Export text alignment using the [editor extension](/conversion/export/docx/editor-extension) or the [REST API](/conversion/export/docx/rest-api). Both handle text alignment identically.

<CodeDemo isPro path="/Extensions/ContentTypeTextAlignmentExport?hideSource=true" />

The exporter reads `node.attrs.textAlign` from each paragraph and heading and maps it back to the DOCX alignment type. No additional configuration is required. If import and editor are configured correctly, export follows automatically.
