Use Tiptap's `@tiptap-pro/extension-export-epub` to export your editor's content as an `.epub` file. This extension integrates EPUB export functionality into your editor by sending the document to the Tiptap convert service for conversion.

You can also use the [REST API](/conversion/export/epub/rest-api) instead if you'd prefer to handle the conversion on your end.

## Install the EPUB Export extension

The Conversion extensions are published in Tiptap's private npm registry. Integrate the extensions by following the [private registry guide](/guides/pro-extensions).

Once done you can install and import the **Export EPUB** extension package.

```bash
npm i @tiptap-pro/extension-export-epub
```

```js
import { ExportEpub } from '@tiptap-pro/extension-export-epub'
```

## Configuring the extension

The `ExportEpub` extension can be configured with an `ExportEpubOptions` (`object`) as an argument to the `configure` method with the following properties:

```ts
import { ExportEpub } from '@tiptap-pro/extension-export-epub'

const editor = new Editor({
  extensions: [
    // Other extensions ...
    ExportEpub.configure({
      endpoint: 'https://api.tiptap.dev/v2/convert',
      token: 'YOUR_TOKEN',
      appId: 'YOUR_APP_ID',
      styleOverrides: {},
      headers: {},
      footers: {},
      pageSize: { width: '21cm', height: '29.7cm' },
      pageMargins: { top: '1cm', bottom: '1cm', left: '1cm', right: '1cm' },
    }),
    // Other extensions ...
  ],
})
```

| Parameter        | Type                     | Description                                    | Default                               |
| ---------------- | ------------------------ | ---------------------------------------------- | ------------------------------------- |
| `endpoint`       | `string`                 | The Tiptap API endpoint for conversion         | `'https://api.tiptap.dev/v2/convert'` |
| `token`          | `string`                 | The Tiptap JWT for authentication              | `''`                                  |
| `appId`          | `string`                 | The Tiptap App ID for authentication           | `''`                                  |
| `styleOverrides` | `Record<string, string>` | Style overrides to apply to the export         | `undefined`                           |
| `headers`        | `HeaderConfig`           | Header configuration for the exported document | `undefined`                           |
| `footers`        | `FooterConfig`           | Footer configuration for the exported document | `undefined`                           |
| `pageSize`       | `PageSize`               | Page size configuration                        | `undefined`                           |
| `pageMargins`    | `PageMargins`            | Page margins configuration                     | `undefined`                           |

### HeaderConfig

| Property            | Type      | Description                                                                                          |
| ------------------- | --------- | ---------------------------------------------------------------------------------------------------- |
| `evenAndOddHeaders` | `boolean` | Whether to use different headers for odd and even pages                                              |
| `default`           | `string`  | The standard default header on every page, or header on odd pages when `evenAndOddHeaders` is active |
| `first`             | `string`  | The header on the first page when the 'Different First Page' option is activated                     |
| `even`              | `string`  | The header on even pages when the `evenAndOddHeaders` option is activated                            |

### FooterConfig

| Property            | Type      | Description                                                                                          |
| ------------------- | --------- | ---------------------------------------------------------------------------------------------------- |
| `evenAndOddFooters` | `boolean` | Whether to use different footers for odd and even pages                                              |
| `default`           | `string`  | The standard default footer on every page, or footer on odd pages when `evenAndOddFooters` is active |
| `first`             | `string`  | The footer on the first page when the 'Different First Page' option is activated                     |
| `even`              | `string`  | The footer on even pages when the `evenAndOddFooters` option is activated                            |

### PageSize

| Property | Type     | Description                                                                                          | Default    |
| -------- | -------- | ---------------------------------------------------------------------------------------------------- | ---------- |
| `width`  | `string` | The width of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).  | `"21cm"`   |
| `height` | `string` | The height of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px). | `"29.7cm"` |

### PageMargins

| Property | Type     | Description                                                                                                        | Default  |
| -------- | -------- | ------------------------------------------------------------------------------------------------------------------ | -------- |
| `top`    | `string` | The top margin of the page. Can be negative. Must be a number followed by a valid unit (cm, in, pt, pc, mm, px).   | `"1cm"`  |
| `bottom` | `string` | The bottom margin of the page. Can be negative. Must be a number followed by a valid unit (cm, in, pt, pc, mm, px).| `"1cm"`  |
| `left`   | `string` | The left margin of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).          | `"1cm"`  |
| `right`  | `string` | The right margin of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).         | `"1cm"`  |

## Export an EPUB file

With the extension installed, you can export your editor's content to `.epub` using the `exportEpub` command.

```ts
/**
 * Export the current document as an EPUB file via the Tiptap convert service.
 *
 * @param options.onExport - Callback function to handle the exported file
 * @example editor.commands.exportEpub({ onExport: (context) => {} })
 */
exportEpub: (options?: { onExport?: (context: ExportContext) => void }) => ReturnType
```

The `onExport` callback receives an `ExportContext` object:

| Property | Type    | Description                                        |
| -------- | ------- | -------------------------------------------------- |
| `blob`   | `Blob`  | The exported EPUB file as a Blob (null on failure)  |
| `error`  | `Error` | The error object (null on success)                  |

```js
import { ExportEpub } from '@tiptap-pro/extension-export-epub'

const editor = new Editor({
  extensions: [
    // Other extensions ...
    ExportEpub.configure({
      token: 'YOUR_TOKEN',
      appId: 'YOUR_APP_ID',
    }),
    // Other extensions ...
  ],
})

function handleExportEpub() {
  editor
    .chain()
    .exportEpub({
      onExport(context) {
        if (context.error) {
          console.error('Export failed:', context.error)
          return
        }

        // Download the EPUB file
        const url = URL.createObjectURL(context.blob)
        const a = document.createElement('a')

        a.href = url
        a.download = 'document.epub'
        a.click()

        URL.revokeObjectURL(url)
      },
    })
    .run()
}
```

### How it works

When you call `exportEpub`, the extension serializes the editor's document to JSON and sends it to the Tiptap convert service along with any configured options (style overrides, page layout, headers, footers). The service converts the document to EPUB and returns it as a binary blob. The `onExport` callback receives this blob, which you can then download or process as needed.

## Support & Limitations

| **Feature**                      | **Support**                                                                |
|----------------------------------|----------------------------------------------------------------------------|
| **Text content**                 | ✓ Basic text, spacing, punctuation                                         |
| **Text formatting**              | ✓ Bold, italic, underline, strikethrough, alignment, line height           |
| **Block elements**               | ✓ Paragraphs, headings (1–6), blockquotes, ordered and unordered lists     |
| **Tables**                       | ✓ Basic structure, header rows, colspan                                    |
| **Links**                        | ✓ Hyperlinks                                                               |
| **Media (Images)**               | ✓ Embedded images, size preserved                                          |
| **Styles**                       | ✓ Font families*, Font colors, font sizes, background colors, line heights |
| **Headers & Footers**            | ✓                                                                          |
| **Math**                         | ✓                                                                          |
| **Sections & Page Breaks**       | ~ In development                                                           |
| **Footnotes & Endnotes**         | ~ In development                                                           |
| **Comments & Revisions**         | ✗                                                                          |
| **Table of Contents**            | ✗                                                                          |
| **Advanced Formatting**          | ✗ Columns, text direction, forms, macros, embedded scripts                 |
| **Metadata**                     | ✗                                                                          |
| **Text Boxes, Shapes, SmartArt** | ✗                                                                          |

<span style={{fontSize: '1rem'}}>* Font families are supported as long as the target font is installed on the operative system when the exported file is opened.</span>
