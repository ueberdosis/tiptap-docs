The [PageBreak extension](/pages/core-concepts/page-break-node) lets users insert explicit page breaks in the editor. When combined with the DOCX export extension, these page breaks can be included in the exported document.

<Callout title="In development" variant="warning">
  DOCX export support for page breaks is currently in development. Exported documents may not yet preserve page break positions. This page will be updated once full support is available.
</Callout>

## Current status

The `@tiptap-pro/extension-pagebreak` extension is fully supported for:

- **Rendering** page breaks in the editor (both standard and [Pages mode](/pages/core-concepts/page-break-node#two-modes-of-operation))
- **Importing** page breaks from `.docx` files via the [DOCX import extension](/conversion/import/docx/page-breaks)
- **Inserting** page breaks manually via `Mod-Enter` or programmatically with `editor.commands.insertPageBreak()`

Export support for page breaks in `.docx` files is under active development.

## Setting up PageBreak with DOCX export

Even though export support is in development, you can already register the PageBreak extension alongside the export extension. Once export support lands, your setup will work automatically.

```js
import { Editor } from '@tiptap/core'
import StarterKit from '@tiptap/starter-kit'
import { ExportDocx } from '@tiptap-pro/extension-export-docx'
import { PageBreak } from '@tiptap-pro/extension-pagebreak'

const editor = new Editor({
  extensions: [
    StarterKit,
    PageBreak,
    ExportDocx.configure({
      onCompleteExport: (result) => {
        const blob = new Blob([result], {
          type: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
        })
        const url = URL.createObjectURL(blob)
        const a = document.createElement('a')
        a.href = url
        a.download = 'export.docx'
        a.click()
        URL.revokeObjectURL(url)
      },
    }),
  ],
})
```
