When exporting to `DOCX`, you can define custom styles that will be applied to the exported document. This is useful when you want to have a consistent look and feel across your documents.

## Per-image overrides with a function

In addition to a static object, the DOCX export accepts a function for `imageOverrides`. The function receives the current image node and returns the overrides for that image, so you can vary the overrides per image based on the node and its attributes. The object form keeps working unchanged, so this is a drop-in addition.

```ts
ExportDocx.configure({
  onCompleteExport: (result) => {
    /* ... */
  },
  // `node` is the image node currently being converted, including its `attrs`.
  imageOverrides: (node) => ({
    // Carry the editor's alt text into Word's accessibility metadata.
    altText: { name: node.attrs?.alt, description: node.attrs?.alt },
    // Derive the rendered size from the dimensions set on the node.
    transformation: {
      width: Number(node.attrs?.width),
      height: Number(node.attrs?.height),
    },
  }),
})
```

The value returned by the function is applied exactly like the static object: it is spread **last**, so the values it returns win over the computed defaults. Returning an empty object leaves every default in place for that image.

<Callout title="Function form is DOCX-only">
  The function form of `imageOverrides` is supported by the DOCX export. Other export formats accept
  the static object form.
</Callout>
