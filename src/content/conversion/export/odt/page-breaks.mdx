The [PageBreak extension](/pages/core-concepts/page-break-node) lets users insert explicit page breaks in the editor. When combined with the ODT export extension, these page breaks can be included in the exported document.

<Callout title="In development" variant="warning">
  ODT export support for page breaks is currently in development. Exported documents may not yet preserve page break positions. This page will be updated once full support is available.
</Callout>

## Current status

The `@tiptap-pro/extension-pagebreak` extension is fully supported for:

- **Rendering** page breaks in the editor (both standard and [Pages mode](/pages/core-concepts/page-break-node#two-modes-of-operation))
- **Importing** page breaks from `.docx` files via the [DOCX import extension](/conversion/import/docx/page-breaks)
- **Inserting** page breaks manually via `Mod-Enter` or programmatically with `editor.commands.insertPageBreak()`

Export support for page breaks in `.odt` files is under active development. The ODT export uses DOCX conversion under the hood, so page break export support will be available once the DOCX export supports it.

## Setting up PageBreak with ODT export

Even though export support is in development, you can already register the PageBreak extension alongside the export extension. Once export support lands, your setup will work automatically.

```js
import { Editor } from '@tiptap/core'
import StarterKit from '@tiptap/starter-kit'
import { ExportOdt } from '@tiptap-pro/extension-export-odt'
import { PageBreak } from '@tiptap-pro/extension-pagebreak'

const editor = new Editor({
  extensions: [
    StarterKit,
    PageBreak,
    ExportOdt.configure({
      appId: 'your-app-id',
      token: 'your-jwt',
      onCompleteExport: (result) => {
        const blob = new Blob([result], { type: 'application/vnd.oasis.opendocument.text' })
        const url = URL.createObjectURL(blob)
        const a = document.createElement('a')
        a.href = url
        a.download = 'export.odt'
        a.click()
        URL.revokeObjectURL(url)
      },
    }),
  ],
})
```
