When you import a DOCX file, the conversion service produces Tiptap JSON containing node types and mark attributes that your editor needs matching extensions to render. Without the right extensions installed, imported content is either missing or unstyled.

ConvertKit is the recommended set of extensions that ensures all importable DOCX content renders correctly in your editor.

## Install

```bash
npm install @tiptap/starter-kit @tiptap/extension-text-style @tiptap/extension-color @tiptap/extension-font-family @tiptap/extension-font-size @tiptap/extension-highlight @tiptap/extension-text-align @tiptap/extension-subscript @tiptap/extension-superscript @tiptap/extension-image @tiptap/extension-table @tiptap-pro/extension-import-docx @tiptap-pro/extension-pagebreak
```

## Configuration

```ts
import StarterKit from '@tiptap/starter-kit'
import { TextStyleKit } from '@tiptap/extension-text-style'
import Color from '@tiptap/extension-color'
import FontFamily from '@tiptap/extension-font-family'
import FontSize from '@tiptap/extension-font-size'
import Highlight from '@tiptap/extension-highlight'
import TextAlign from '@tiptap/extension-text-align'
import Subscript from '@tiptap/extension-subscript'
import Superscript from '@tiptap/extension-superscript'
import { Image } from '@tiptap/extension-image'
import { TableKit } from '@tiptap/extension-table'
import { PageBreak } from '@tiptap-pro/extension-pagebreak'
import { ImportDocx } from '@tiptap-pro/extension-import-docx'

const editor = new Editor({
  extensions: [
    StarterKit,
    TextStyleKit,
    Color,
    FontFamily,
    FontSize,
    Highlight.configure({ multicolor: true }),
    TextAlign.configure({ types: ['paragraph', 'heading'] }),
    Subscript,
    Superscript,
    Image,
    TableKit,
    PageBreak,
    ImportDocx.configure({
      appId: 'your-app-id',
      token: 'your-jwt',
      imageUploadConfig: {
        url: 'https://your-image-upload-endpoint.com',
      },
    }),
  ],
})
```

## What each extension handles

| Extension | What it renders | Without it |
| --- | --- | --- |
| `StarterKit` | Paragraphs, headings, bold, italic, underline, strike, code, blockquotes, lists, horizontal rules, hard breaks, links | Basic document structure is missing |
| `TextStyleKit` | Base mark for text color, font family, and font size | Color, font, and size attributes have nowhere to attach |
| `Color` | Text color from `<w:color>` | Text appears without color |
| `FontFamily` | Font family from `<w:rFonts>` | All text renders in the editor's default font |
| `FontSize` | Font size from `<w:sz>` | All text renders at the editor's default size |
| `Highlight.configure({ multicolor: true })` | Highlight and shading from `<w:highlight>` and `<w:shd>` | Highlighted text appears unstyled. Without `multicolor: true`, all highlights render as yellow. |
| `TextAlign.configure({ types: ['paragraph', 'heading'] })` | Text alignment (left, center, right, justify) | All text is left-aligned regardless of the source document |
| `Subscript` | Subscript text from `<w:vertAlign val="subscript">` | Subscript text renders inline (e.g., Hâ‚‚O appears as H2O) |
| `Superscript` | Superscript text from `<w:vertAlign val="superscript">` | Superscript text renders inline (e.g., xÂ˛ appears as x2) |
| `Image` | Inline images | Images are not rendered |
| `TableKit` | Tables with merged cells, column widths, header rows | Tables are not rendered |
| `PageBreak` | Page break nodes | Page breaks are not rendered. See the [invalid schema guide](/guides/invalid-schema). |

<Callout variant="info" title="Optional: Pages extension for paginated layout">
  For documents that rely on page layout (headers, footers, page numbers, page size), add the [Pages extension](/pages/getting-started/overview). Without it, page breaks render as visual dividers but content does not flow across pages.
</Callout>

## What ConvertKit does not cover

Some imported DOCX content requires custom handling regardless of which extensions are installed:

- **Footnote and endnote references** â€” The import produces `footnoteReference` and `endnoteReference` nodes, but no editor extension exists to render them. See [Footnotes and endnotes](/conversion/content-types/structures-and-media/footnotes-endnotes).
- **Paragraph spacing, indentation, borders** â€” These are imported as node attributes but require a [custom extension](/editor/extensions/custom-extensions/extend-existing) to render. See [Paragraphs](/conversion/content-types/text-and-formatting/paragraphs).
- **Table of Contents** â€” Imported as a `tableOfContents` node that requires a custom extension. See [Supported features](/conversion/getting-started/feature-support-matrix).
- **Word styles** â€” Style names are used for structural detection (headings, blockquotes) but visual styling is not preserved. See [Word styles](/conversion/content-types/structures-and-media/word-styles).

For guidance on styling your editor to match Word's appearance, including CSS examples and how to extend extensions for unsupported attributes, see the [Styling converted content](/conversion/getting-started/guides/styling-converted-content) guide.
