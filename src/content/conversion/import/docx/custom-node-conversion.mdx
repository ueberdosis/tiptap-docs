The `@tiptap-pro/extension-import-docx` extension allows you to define how custom nodes in a DOCX file should be mapped back to your Tiptap schema during import.

## Import custom nodes from .docx

<CodeDemo src="https://feature-beta-conversion-extensions--tiptap-pro.netlify.app/preview/Extensions/ImportDocxCustomNode" />

When importing a DOCX file, you can map custom Word paragraph styles back to your Tiptap node types using the `prosemirrorNodes` option.

```ts
import { ImportDocx } from '@tiptap-pro/extension-import-docx'

// ... inside your Editor or useEditor setup:
ImportDocx.configure({
  appId: 'your-app-id',
  token: 'your-jwt',
  // ATTENTION: This is for demo purposes only
  endpoint: 'https://your-endpoint.com',
  imageUploadCallbackUrl: 'https://your-endpoint.com/image-upload',
  // ProseMirror custom node mapping
  prosemirrorNodes: {
    Hintbox: 'hintbox',
  },
}),
```

The `prosemirrorNodes` option takes a key-value map where:

- The **key** is the Word paragraph style name as it appears in the DOCX — in this example `Hintbox`. This must match exactly, including casing.
- The **value** is the Tiptap node type to map it to — in this example `hintbox`.

Whenever the DOCX contains a paragraph with the `Hintbox` style, it will be converted to a `hintbox` node in your Tiptap editor.

<Callout title='DOCX, "prosemirrorNodes" and "prosemirrorMarks"' variant="info">
  Please note that the `prosemirrorNodes` and `prosemirrorMarks` options will only work if
  you&apos;re importing a `.docx` file. If you&apos;re importing another type of file, eg: an `.odt`
  file, the `/import` endpoint will be used instead of the `/import-docx` endpoint, and the
  `prosemirrorNodes` and `prosemirrorMarks` options will not be available, and therefore you&apos;d
  need to rely on the [custom node and mark mapping
  API](/conversion/import/docx/rest-api#custom-node-and-mark-mapping) for those endpoints.
</Callout>

## Round-tripping custom nodes

If you exported this DOCX using the `ExportDocx` extension and want to import it back into Tiptap, the key in `prosemirrorNodes` must match the `id` you set in `styleOverrides.paragraphStyles` on the export side. The key is case-sensitive and must match exactly.

<Callout title="Using ExportDocx for round-trips" variant="info">
  See [Export custom nodes to .docx](/conversion/export/docx/custom-node-conversion#round-tripping-custom-nodes) for details on
  how to define custom styles on the export side.
</Callout>
