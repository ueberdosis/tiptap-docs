The Tiptap CLI helps you scaffold editors, install UI components or primitives, and configure Tiptap faster. It removes much of the manual work so you can start building editor features faster.

It’s designed for Vite or Next.js projects. If you use another framework, you’ll need to integrate Tiptap manually from the [open-source repository](https://github.com/ueberdosis/tiptap-ui-components).

## Usage

```bash
npx @tiptap/cli [command] [options]
```

## Commands

### `init`

Initialize your project and install dependencies. This command sets up your project configuration and adds any specified components.

```bash
npx @tiptap/cli init
```

#### Arguments

- components: the components to add

#### Options

```bash
Usage: @tiptap/cli init [options] [components...]

Options:
-f, --framework <framework>   The framework to use (e.g., next, vite)
-f, --force                   Force overwrite of existing configuration
-c, --cwd <cwd>               The working directory (defaults to current directory)
-s, --silent                  Mute output
--src-dir                     Default: Use the src directory when creating a new project
--no-src-dir                  Do not use the src directory when creating a new project
```

### `add`

Add one or more components to your project.

```bash
npx @tiptap/cli add [components]
```

#### Example

```bash
npx @tiptap/cli add button
```

#### Options

```bash
Usage: @tiptap/cli add [options] [components...]

Arguments

    components          the components to add

Options

    -c, --cwd <cwd>     the working directory (defaults to current directory)
    -p, --path <path>   the path to add the component to
    -s, --silent        mute output
```

### `info`

Get information about your project

```bash
npx @tiptap/cli info [options]
```

#### Example

```bash
npx @tiptap/cli info -c /path/to/project
```

#### Options

```bash
Usage: @tiptap/cli info [options]

Options

    -c, --cwd <cwd>     the working directory (defaults to current directory)
```

### `login`

Log in to your Tiptap registry account

```bash
npx @tiptap/cli login [options]
```

#### Example

```bash
npx @tiptap/cli login -e my-email -p my-password --write-config
```

#### Options

```bash
Usage: @tiptap/cli login [options]

Options

    -e, --email <email>   your Tiptap registry email
    -p, --password <password>   your Tiptap registry password
    --write-config   write the login information to the package manager config file
    -c, --cwd <cwd>     the working directory (defaults to current directory)
```

### `logout`

Log out of your Tiptap registry account

```bash
npx @tiptap/cli logout
```

#### Example

```bash
npx @tiptap/cli logout
```

### `status`

Show the status of your Tiptap registry account

```bash
npx @tiptap/cli status [options]
```

#### Example

```bash
npx @tiptap/cli status
```

#### Options

```bash
Usage: @tiptap/cli status [options]

Options

    -c, --cwd <cwd>     the working directory (defaults to current directory)
```
