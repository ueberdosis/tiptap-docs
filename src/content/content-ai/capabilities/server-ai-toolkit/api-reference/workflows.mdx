The Server AI Toolkit workflow API is split into three endpoint groups:

1. **Workflow definitions** return the workflow prompt and output schema.
2. **Workflow read endpoints** prepare the document state for the model.
3. **Workflow execute endpoints** apply the generated workflow output to the document.

## Workflow definition endpoints

Use these endpoints before calling your model:

- `POST /v3/ai/toolkit/workflows/edit`
- `POST /v3/ai/toolkit/workflows/insert-content`
- `POST /v3/ai/toolkit/workflows/proofreader`
- `POST /v3/ai/toolkit/workflows/threads`

Each endpoint accepts:

- `format` (`'json' | 'shorthand'`)

`proofreader` currently supports `format: 'shorthand'` only.

Each endpoint returns:

- `systemPrompt` (`string`): Prompt text to prepend to your model call
- `outputSchema` (`object`): JSON schema for structured model output

## Workflow read endpoints

Use workflow read endpoints to prepare the exact document state your model needs.

### `POST /v3/ai/toolkit/read/read-document`

Reads a document range and returns workflow-ready content.

- Supports `document` or `experimental_documentOptions`
- Accepts `range`, `format`, `sessionId`, `reviewOptions`, and `schemaAwarenessData`

### `POST /v3/ai/toolkit/read/read-selection`

Reads the current selection and returns either selection data or an explicit empty state.

- Supports `document` or `experimental_documentOptions`
- Accepts `range`, `format`, `sessionId`, `reviewOptions`, and `schemaAwarenessData`
- Returns `output.isEmpty = true` when the selection is empty
- Returns `content`, `nodeHashes`, `nodeRange`, and `prompt` when `isEmpty = false`

### `POST /v3/ai/toolkit/read/threads`

Reads all threads and comments from a Tiptap Cloud document.

- Requires `experimental_documentOptions.documentId`
- Accepts `format` and `schemaAwarenessData`

## Workflow execute endpoints

Use execute endpoints after your model returns structured workflow output.

### `POST /v3/ai/toolkit/execute-workflow/edit`

Applies edit operations to a document.

- Supports `document` or `experimental_documentOptions`
- Accepts `input`, `format`, `sessionId`, `reviewOptions`, and `schemaAwarenessData`

### `POST /v3/ai/toolkit/execute-workflow/insert-content`

Replaces content at a range with generated content.

- Supports `document` or `experimental_documentOptions`
- Accepts `input`, `range`, `format`, `sessionId`, `reviewOptions`, and `schemaAwarenessData`
- Returns `output.range` with the inserted content range when available

### `POST /v3/ai/toolkit/execute-workflow/proofreader`

Applies proofreader operations to a document.

- Supports `document` or `experimental_documentOptions`
- Accepts `input`, `format: 'shorthand'`, `sessionId`, `reviewOptions`, and `schemaAwarenessData`
- Use [`reviewOptions`](/content-ai/capabilities/server-ai-toolkit/api-reference/review-options) to
  apply corrections as tracked changes

### `POST /v3/ai/toolkit/execute-workflow/threads`

Applies thread and comment operations.

- Requires `experimental_documentOptions.documentId`
- Accepts `input`, `format`, `schemaAwarenessData`, and optional
  `experimental_commentsOptions`

## Common workflow options

### `format`

- `'json'`: Richer document payloads
- `'shorthand'`: More token-efficient format for model prompts

### `sessionId`

Pass a `sessionId` when a workflow spans multiple requests and should reuse the same server-side
session state.

### `reviewOptions`

Use review options to control how edits are applied:

- `mode: 'disabled'`: Apply changes directly
- `mode: 'trackedChanges'`: Write suggestions as tracked changes

## Document sources

Most workflow read and execute endpoints accept one document source:

- `document`: Inline Tiptap JSON
- `experimental_documentOptions`: A Tiptap Cloud document reference

Thread-related endpoints require `experimental_documentOptions` because comments live on the
Document Server.
